class (name[, base[, is_abstract]])
class utility for creating object oriented inheritance hierarchies of tree nodes

Parameters:

  • name string The name of the new class to create
  • base tab The Base class of the new class to inherit from (optional)
  • is_abstract bool Defines whether or not this class should be directly used as a node type in a behavior tree (optional)

Returns:

    tab New class type

Loads an owlbt format json file, parses it into a table, and converts it into the behaviors mod format.

Parameters:

  • path string The file path of the owlbt json file to load.

Returns:

    tab The parsed and converted behavior tree definition table.

Instantiates nodes from a behavior tree definition table using the registered node types.

Parameters:

  • source tab The behavior tree defnition table.

Returns:

    node The root node of the instantiated node tree.

Loads a behavior tree definition table, converts it to behavior mod format, and registers it as a module subtree.

Parameters:

  • name string The name to register the subtree module as.
  • path string The file path to the owlbt json file defining the behavior tree.

Registers a subtree module that can be used as a single subtree action node for modularity and simpler trees with complex behaviors.

Parameters:

  • name string The name of the subtree module to register
  • source tab The parsed (and already converted from owlbt) subtree definition table.

Tables

states
Enum table of possible node states

Fields:

  • RUNNING The node state when it is not done running
  • FAILED The node state when it fails
  • SUCCESS The node state when it succeeds
parallel_strategy
Enum table of possible parallel strategies for use with the parallel node class

Fields:

  • FAIL_CERTAIN This makes parallel nodes fail when the number of failures reaches the target count.
  • PASS_CERTAIN This makes parallel nodes succeed when the number of success reaches the target count.

tree:run (any)
Runs one step of the tree starting with the root node which runs it's children until the entire tree is processed.

Parameters:

  • any Any parameters are passed down the tree of nodes to each call of the child nodes' run methods.

Returns:

    A string representing the enum behaviors.states of "running", "success", or "failed".
tree:is_finished ()
Checks if the tree is finished processing it's nodes. For most cases the tree is reset and run again once it finishes.

Returns:

    bool Whether the tree is finished running or not
